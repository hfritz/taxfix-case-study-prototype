# Specs

Specs are living agreements. They clarify the problem, solution, constraints, and expected outcomes.

Use the templates in `specs/templates/` as starting points. Remove sections that do not apply, but do not skip the thinking.

## Recommended Naming

```txt
specs/
  2026-05-12-example-feature/
    product-spec.md
    technical-spec.md
    implementation-plan.md
    decisions/
      001-example-decision.md
```

## Spec Review Checklist

- The target user is clear.
- The problem is specific.
- Goals and non-goals are explicit.
- Success metrics are measurable or honestly marked as unknown.
- Key flows and edge cases are described.
- Major risks and assumptions are visible.
- The technical approach is understandable.
- The release and learning plan are practical.
