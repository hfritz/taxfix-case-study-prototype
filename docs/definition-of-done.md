# Definition Of Done

A piece of work is done when:

- It solves the stated problem within agreed scope.
- Core flows and edge cases are verified.
- Relevant tests or manual checks pass.
- User-facing states are complete.
- Metrics or observability are in place where needed.
- Documentation and decision records are updated if behavior or direction changed.
- Rollout and rollback are understood.
- Known risks are accepted by the owner.
