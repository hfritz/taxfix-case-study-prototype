# Pre-Build Checklist

Before implementation starts:

- [ ] Problem is clear.
- [ ] Target user is clear.
- [ ] Goals and non-goals are explicit.
- [ ] Success metrics are defined or honestly marked as unknown.
- [ ] Scope is small enough to build.
- [ ] Technical risks are understood.
- [ ] Edge cases are listed.
- [ ] Test or verification approach is known.
- [ ] Rollout and rollback are considered.
