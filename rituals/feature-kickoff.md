# Feature Kickoff

Use this before building a meaningful feature.

## Checklist

- [ ] Product spec exists.
- [ ] Goals and non-goals are clear.
- [ ] User flow is described.
- [ ] Success metrics and guardrails are defined.
- [ ] Technical approach is agreed.
- [ ] Risks and edge cases are visible.
- [ ] Implementation plan exists.
- [ ] Verification plan exists.
- [ ] Rollout path is known.

## Anti-Patterns

- Building because the idea sounds obvious.
- Treating metrics as an afterthought.
- Starting with implementation details before the user problem is clear.
- Hiding unresolved questions inside vague scope.
