# Project Kickoff

Use this when starting a new project from the foundation repo.

## Checklist

- [ ] Create or update `README.md`.
- [ ] Create a problem brief.
- [ ] Draft the product spec.
- [ ] Identify target users.
- [ ] Define goals, non-goals, and success metrics.
- [ ] List major assumptions.
- [ ] Decide whether a technical spec is needed.
- [ ] Identify initial milestones.
- [ ] Record important decisions.

## Kickoff Questions

- Who is this for?
- What problem are we solving?
- Why now?
- What does success look like?
- What will we intentionally not do?
- What is the smallest useful version?
- What evidence would change our mind?
