# Pre-Release Checklist

Before release:

- [ ] Core user flows pass.
- [ ] Empty, loading, error, and success states work.
- [ ] Relevant tests pass.
- [ ] Analytics or observability is in place.
- [ ] Accessibility basics are checked.
- [ ] Rollout plan is clear.
- [ ] Rollback plan is clear.
- [ ] Support, ops, or stakeholder communication is prepared if needed.
- [ ] Known risks are accepted.
