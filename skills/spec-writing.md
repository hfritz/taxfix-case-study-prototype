# Skill: Spec Writing

## Use When

An idea is promising but not yet clear enough to build.

## Workflow

1. Restate the idea in plain language.
2. Identify the target user.
3. Describe the current pain or limitation.
4. Separate goals from proposed solution.
5. Define non-goals.
6. Propose success metrics and guardrails.
7. Capture assumptions and open questions.
8. Suggest the smallest useful validation path.

## Output

A product spec or problem brief that is specific, measurable, and ready for review.

## Quality Check

The spec should make it easier to say yes, no, or not yet.
