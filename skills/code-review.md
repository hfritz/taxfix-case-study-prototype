# Skill: Code Review

## Use When

Code changes need review before merge or release.

## Workflow

1. Understand the intended behavior.
2. Inspect the changed files.
3. Look for bugs, regressions, missing edge cases, and missing tests.
4. Check whether the implementation fits local patterns.
5. Evaluate observability, security, performance, and maintainability where relevant.
6. Lead with findings ordered by severity.

## Output

A review focused on actionable risks, with file and line references when available.

## Quality Check

The review should catch meaningful problems, not merely express preferences.
