# Skill: Technical Planning

## Use When

A product spec needs to become an implementation plan.

## Workflow

1. Read the product spec.
2. Inspect the existing system.
3. Identify affected modules, data, APIs, and dependencies.
4. List key edge cases and failure modes.
5. Propose the simplest viable architecture.
6. Define milestones and verification steps.
7. Call out tradeoffs and rollback options.

## Output

A technical spec and implementation plan that are small enough to execute and complete enough to review.

## Quality Check

A competent engineer should be able to start implementation without guessing at the major decisions.
