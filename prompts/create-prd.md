# Prompt: Create A Product Spec

Use the context below to create a concise product spec.

Ask clarifying questions only if the missing information would materially change the spec. Otherwise, make assumptions explicit.

Include:

- Summary.
- Target user.
- Problem.
- Goals.
- Non-goals.
- Key flows.
- Success metrics.
- Risks.
- Open questions.

Context:

```txt
[Paste context here]
```
