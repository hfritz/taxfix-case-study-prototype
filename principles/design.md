# Design Principles

Design is how the product communicates, behaves, and earns trust.

## Clarity Before Decoration

The interface should make the next useful action obvious. Visual styling should support comprehension, not compete with it.

## Fit The Domain

Design choices should match the product's context. Operational tools should feel efficient and scannable. Consumer experiences may carry more emotion, guidance, and delight.

## Respect User Attention

Reduce cognitive load. Avoid unnecessary choices, ambiguous labels, hidden state, and flows that require users to remember too much.

## Make States Explicit

Every meaningful interaction should account for loading, empty, error, success, disabled, and permission states.

## Design For Repeated Use

Many products are not used once. Optimize common workflows for speed, predictability, and confidence.

## Accessibility Is Product Quality

Accessible products are better products. Use semantic structure, sufficient contrast, keyboard support, readable text, and clear feedback.

## Words Are Interface

Labels, errors, empty states, and confirmations should be specific, humane, and useful. Avoid vague encouragement when direct guidance would help.
