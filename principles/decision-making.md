# Decision-Making Principles

## Make The Decision Type Explicit

Name whether a decision is reversible, strategic, technical, product, design, legal, or operational.

## Prefer Evidence, But Do Not Wait Forever

Use the best available evidence. When evidence is incomplete, state the assumption and define how it will be tested.

## Optimize For The System

Avoid local decisions that damage the broader user experience, operating model, business, or platform.

## Record Meaningful Tradeoffs

If a decision changes architecture, product scope, user experience, or business risk, create a decision record.

## Revisit Decisions With New Evidence

A good decision can become outdated. Define what would cause the team to revisit it.
